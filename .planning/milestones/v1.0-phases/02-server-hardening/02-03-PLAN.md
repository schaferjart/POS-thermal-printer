<objective>
Add SIGTERM/SIGINT graceful shutdown handler to the print server and fix the portrait pipeline config mutation bug.

Purpose: The server currently uses atexit for cleanup, which Flask's dev server may not trigger on SIGTERM from systemd. This means mDNS stays registered and the USB printer connection is never properly closed on `systemctl stop`. The portrait pipeline mutates the shared _config dict, causing state leaks between requests.

Output: Clean shutdown on SIGTERM (mDNS deregistered, printer closed), and portrait pipeline that never mutates shared config.
</objective>

**1. Add graceful_shutdown to print_server.py.**

Add `import signal` at the top of print_server.py (near the other stdlib imports).

Add the graceful_shutdown function before main():
```python
def graceful_shutdown(signum, frame):
    """Clean up mDNS and printer connection on SIGTERM/SIGINT."""
    logger.info("Received signal %s, shutting down...", signum)
    if _zeroconf:
        try:
            _zeroconf.unregister_all_services()
            _zeroconf.close()
            logger.info("mDNS deregistered")
        except Exception:
            pass
    if _printer:
        try:
            _printer.close()
            logger.info("Printer connection closed")
        except Exception:
            pass
    sys.exit(0)
```

In main(), BEFORE `app.run()`, register the signal handlers:
```python
signal.signal(signal.SIGTERM, graceful_shutdown)
signal.signal(signal.SIGINT, graceful_shutdown)
```

Keep the existing atexit.register(cleanup) in register_mdns() as a belt-and-suspenders fallback for clean interpreter exit.

CRITICAL: Do NOT acquire _print_lock in the signal handler. The signal may arrive while a print job holds the lock, which would deadlock. Just close the printer and mDNS directly -- the ongoing print job will fail but the process is exiting anyway.
